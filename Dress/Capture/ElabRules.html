    Elaboration Rules for Blueprint Declarations #

    This module provides elab_rules that intercept declarations with @[blueprint] attributes and capture highlighting after elaboration while info trees are still available.

    Architecture #

    We use elab_rules to intercept declarations that have @[blueprint]. After standard elaboration completes (but while info trees still exist), we capture the highlighting and store it in the environment extension.

    The key insight is that elab_rules run before the standard elaborators, and we can call the standard elaborator explicitly, then capture highlighting afterward.

                        Elaboration Rules #

                        We use elab_rules to intercept declarations with @[blueprint] attribute. These intercept declarations and capture highlighting after elaboration.

                        We use scoped rules so they only apply when Dress is imported. The rules check the inCaptureHook flag to prevent infinite recursion.
