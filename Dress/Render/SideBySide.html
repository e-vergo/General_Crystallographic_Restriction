    Side-by-Side Rendering #

    Core types and functions for rendering side-by-side LaTeX + Lean code displays. This module provides a unified rendering API used by both blueprint pages and paper generation.

    source
    structure Dress.Render.SbsData :
    Type

    Data needed for rendering a side-by-side display

    • id : String

      HTML element ID

    • label : String

      Display label (e.g., "4.1.1")

    • displayNumber : Option String

      Optional custom display number

    • envType : String

      Environment type ("theorem", "lemma", "definition", etc.)

    • status : Architect.NodeStatus

      Node status for status indicator

    • statementHtml : String

      LaTeX statement (already rendered to HTML)

    • proofHtml : Option String

      LaTeX proof (already rendered to HTML)

    • signatureHtml : Option String

      Lean signature with syntax highlighting

    • proofBodyHtml : Option String

      Lean proof body with syntax highlighting

    • hoverData : Option String

      JSON data for hover tooltips

    • declNames : Array Lean.Name

      Fallback declaration names
